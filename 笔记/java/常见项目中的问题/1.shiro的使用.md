## 概述
```
    拥有权限的用户，才可以访问某种权限，想要访问，就必须有权限。
    验证合法性：一般是密码、指纹等
    关键对象：
        主体：理解为用户，可以是程序，需要访问资源，需要对主体进行身份验证
        身份信息：一个主体多个身份信息，但是有一个主的身份信息
        凭证信息：可能是密码、证书、指纹。

    用户授权：认证结束之后，系统会对用户访问资源进行控制，用户具有资源的访问权限当可以控制

    过程：who 对 which  进行 how操作
    who：主体，认证之后进行访问控制
    what：资源，具有权限可以访问资源
    how:就是什么样的权限
```
## 权限模型
```
    主体、资源、权限、角色、角色和关系、主体和角色的关系

```
## 权限的方式
```
    基于角色
        实现就是你是谁，你有什么样的权限
        可扩展性不强，有一天另一个角色也要权限，又需要进行修改。

    基于权限
          你有权限，你来访问，不管你是谁。
  

```
## 权限的解决方案
```
    粗粒度
        粗粒度可能就是一个页面
    细粒度
        细粒度就是对某一个按钮

    如何实现：
        粗粒度可以直接通过拦截器进行处理，因为他的访问比较具有共性
        细粒度  共性弱，一般会在service中进行处理。   

```
