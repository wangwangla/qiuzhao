## 对象入门  
```

    1.面向对象编程（OOP）
    优点：更快更廉价的开发与维护，建模更简单，生成清晰，易于维护的设计方案
    缺点：设计具有挑战性，一般人无法设计出更好的，只能通过使用别人的来解决自己的问题。

    2.解决一个问题，我们创建出一种数据类型，但是在面向对象中，我们设计成为class的类，通过创建对象，来进行操作。

    3.面向对象编程分为两种人：一种是类创建者，一类是类的时候，类创造者将自己需要开放的部分，开放，其他部分将其隐藏起来。

    4.灵活性：
     新类的成员变量通常设置为私有，好处就是可以安全的修改它。
    
    5.继承：继承可以是代码重用性增强，但是这样将会使得设计更加笨做，增加复制性。
    继承可以实现代码的重用，减少重复书写，但是这种重用不等于克隆，继承是父类发生了改变，那么子类也会发生改变， 但是克隆是可以实现在克隆后与父类没有任何瓜葛。

    继承中private是不可以访问的，假如子类不对父类进行重写，那么就将父类直接繁衍到子类中，可以通过两种方式进行分开：
        （1）子类有自己的心功能，父类不可以满足需求的时候
        （2）对父类进行改善，增加新东西。
